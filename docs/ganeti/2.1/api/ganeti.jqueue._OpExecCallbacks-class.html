NotifyStart(self)

source code 

Mark the opcode as running, not lock-waiting.

This is called from the mcpu code as a notifier function, when the LU is finally about to start the Exec() method. Of course, to have end-user visible results, the opcode must be initially (before calling into Processor.ExecOpCode) set to OP_STATUS_WAITLOCK.
